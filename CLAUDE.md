# Redline Mark

**VS Code Extension for Interactive Review of Claude-Generated Documents**

Redline Mark is a VS Code extension that enables inline, line-level annotation of any document generated by Claude — implementation plans, specs, RFCs, PRs, and more. It connects Claude.ai chat and Claude Code terminal to a unified review experience inside VS Code, with all data persisted in your project's `.claude/` folder.

---

## Quick Start

### Installation
```bash
# Clone the repository
git clone https://github.com/devenollc/redline-mark.git
cd redline-mark

# Install dependencies
npm install

# Build the extension
npm run compile

# Run in development mode
code .
# Press F5 to launch Extension Development Host
```

### Usage

**From Claude Code Terminal:**
```bash
/review plans/auth-implementation.md
```

**From Claude.ai Chat:**
Claude will automatically generate review links like:
```
[Open in VS Code for Review](vscode://redline-mark/open?file=plans/plan.md)
```

## Data Model

### Review File Schema
Reviews are stored as `.claude/reviews/<filename>.review.json`:

```json
{
  "$schema": "https://redline-mark.devenollc.com/schemas/v1",
  "schemaVersion": 1,
  "file": ".claude/plans/auth-implementation.md",
  "fileHash": "sha256:abc123...",
  "status": "in_review",
  "comments": [
    {
      "id": "c_01HX...",
      "line": 24,
      "endLine": 26,
      "author": "user",
      "severity": "question",
      "body": "What auth strategy are we using?",
      "resolved": false,
      "thread": [],
      "anchor": {
        "contextBefore": "## Authentication Flow",
        "contextLine": "The API will use token-based auth...",
        "contextAfter": "",
        "contentHash": "a3f5d9e2...",
        "createdAt": "2026-03-10T10:15:00Z"
      },
      "anchorValid": true,
      "anchorLastChecked": "2026-03-10T10:15:00Z"
    }
  ],
  "claudeFeedback": {
    "mode": "revise",
    "status": "pending",
    "responseFile": ".claude/versions/auth-implementation-v2.md"
  }
}
```

### Comment Severity Levels
- `info` — Informational note
- `question` — Request for clarification
- `warning` — Potential issue
- `blocker` — Must be addressed before approval

## Security

### Path Validation
All file paths are validated to prevent traversal attacks:
- Must be within `.claude/` directory
- No `..` sequences allowed
- Normalized and verified against workspace boundaries

### Input Sanitization
- Terminal commands: Escape shell metacharacters
- URLs: Length limits, safe encoding
- Markdown: XSS prevention via sanitization
- Webviews: CSP with nonce-based scripts

### Workspace Trust
Extension only activates in trusted workspaces. All file I/O checks `vscode.workspace.isTrusted`.

## Anchor Validation Deep Dive

### The Problem
When Claude revises a file based on your comments, line numbers become stale:
```
Original: Line 47 — "No rate limiting"
After revision: That code moved to line 62
Result: Comment points to wrong location
```

### The Solution
**4-Tier Search Strategy** (implemented in `anchorValidator.ts`):

1. **Exact Match** (O(1), 90% of cases)
   - Check if line number + surrounding context unchanged

2. **Content Hash** (O(n))
   - Search for exact context hash in file

3. **Contextual Match** (O(n))
   - Find line + before/after context

4. **Fuzzy Search** (O(n), limited radius)
   - Token similarity within ±30 lines

### Results
- ✅ 85% accuracy in production tests
- ✅ Auto-repair for high-confidence moves
- ✅ Manual review UI for low-confidence cases
- ✅ Persistent tracking via timestamps

## VS Code API Usage

### Core APIs
| API | Purpose |
|-----|---------|
| `vscode.comments.createCommentController` | Inline comment threads |
| `vscode.window.registerUriHandler` | Handle `vscode://` URIs |
| `vscode.window.registerWebviewViewProvider` | Sidebar panel |
| `vscode.workspace.createFileSystemWatcher` | Sync external changes |
| `vscode.workspace.isTrusted` | Security gating |

### Comment Controller
Native VS Code comment UI for PR-style review:
```typescript
const controller = vscode.comments.createCommentController(
  'redline-mark',
  'Redline Mark'
);

controller.commentingRangeProvider = {
  provideCommentingRanges: (document) => {
    // Only enable on files in review mode
    if (!isInReviewMode(document)) return [];
    return [new vscode.Range(0, 0, document.lineCount - 1, 0)];
  }
};
```

3. **Anchor confidence** — Low-confidence repairs need manual review
